# 7. Верификация и валидация (V&V) — ParkPoint

## Обозначения

### Метод проверки:
- **T** — тест (функциональный, интеграционный, нагрузочный)
- **A** — анализ (метрики, SLA, отчёты)
- **I** — инспекция (архитектура, конфигурация, UI, БД)
- **D** — демонстрация (сквозной пользовательский сценарий)

# 7.2 Тест-кейсы

---

TC-01 — Въезд автомобиля через LPR
Цель

Проверка автоматического въезда автомобиля на парковку с использованием системы распознавания номеров (LPR).

Предусловия
Камера распознавания номеров (LPR) активна и подключена к системе
Автомобиль зарегистрирован в системе ParkPoint
Номер автомобиля присутствует в базе данных пользователей
Шаги
Автомобиль подъезжает к въездному шлагбауму
Камера фиксирует номер автомобиля
Система выполняет распознавание номера (LPR)
Система сверяет номер с базой зарегистрированных пользователей
При успешном совпадении формируется команда на открытие шлагбаума
Система фиксирует событие въезда в журнале
Ожидаемый результат
Шлагбаум открывается автоматически
В базе данных создаётся запись о въезде (номер авто, время, точка въезда)
Статус автомобиля изменяется на «на парковке»
Событие фиксируется в журнале событий (audit log)
TC-02 — Расчёт стоимости парковки
Цель

Проверка корректности автоматического расчёта стоимости парковки.

Предусловия
Автомобиль находится на парковке
Зафиксировано время въезда
Настроен тариф (например, 100 руб/час)
Шаги
Пользователь завершает парковочную сессию
Система фиксирует время выезда
Вычисляется длительность пребывания
Применяется тарифная модель
Рассчитывается итоговая стоимость
Сумма отображается пользователю
Ожидаемый результат
Стоимость рассчитана корректно
Формула расчёта соответствует: время × тариф
Результат совпадает с тарифной политикой системы
TC-03 — Оплата парковки через приложение
Цель

Проверка процесса оплаты парковки через мобильное приложение.

Предусловия
Существует завершённая парковочная сессия
Подключён платёжный шлюз
Пользователь авторизован
Шаги
Пользователь открывает мобильное приложение
Переходит к активной парковочной сессии
Выбирает действие «Оплатить»
Подтверждает платёж
Система отправляет запрос в платёжный шлюз
Получает ответ о статусе транзакции
Ожидаемый результат
Статус сессии изменяется на «Оплачено»
Парковочная сессия закрывается
Пользователь получает уведомление об успешной оплате
Транзакция фиксируется в журнале событий
TC-04 — Резервирование парковочного места
Цель

Проверка функции предварительного бронирования парковочного места.

Предусловия
Доступны свободные парковочные места
Пользователь авторизован
Шаги
Пользователь открывает карту парковки
Выбирает доступное место
Указывает время бронирования
Подтверждает резервирование
Система фиксирует бронь
Ожидаемый результат
Парковочное место закреплено за пользователем
Статус места изменяется на «reserved»
Бронь отображается в системе
TC-05 — Контроль занятости парковочных мест
Цель

Проверка корректного отображения занятости парковки в реальном времени.

Предусловия
Камеры или датчики занятости активны
Парковка частично заполнена
Шаги
Несколько автомобилей заезжают на парковку
Система фиксирует занятость мест
Обновляется карта парковки
Пользователь открывает интерфейс просмотра
Ожидаемый результат
Карта парковки обновляется в реальном времени
Количество свободных и занятых мест отображается корректно
Задержка обновления отсутствует или минимальна
TC-06 — Отказ платёжного сервиса
Цель

Проверка поведения системы при недоступности платёжного шлюза.

Предусловия
Платёжный сервис недоступен (симуляция отказа)
Активная парковочная сессия существует
Шаги
Пользователь завершает парковку
Система отправляет запрос на оплату
Платёжный сервис не отвечает
Система фиксирует ошибку
Переводит оплату в статус «ожидание»
Ожидаемый результат
Сессия не теряется
Статус оплаты изменяется на «ожидание»
Пользователь получает уведомление о задержке
TC-07 — Изменение тарифов администратором
Цель

Проверка изменения тарифной политики парковки.

Предусловия
Пользователь с ролью администратора авторизован
Шаги
Администратор открывает панель управления
Выбирает раздел тарифов
Изменяет значение тарифа
Сохраняет изменения
Ожидаемый результат
Новый тариф применяется ко всем новым парковочным сессиям
Старые сессии не пересчитываются
TC-08 — Журнал событий (audit log)
Цель

Проверка неизменяемости журнала событий.

Предусловия
В системе выполнены операции (въезд, оплата, выезд)
Шаги
Открыть журнал событий
Проверить записи
Попытаться изменить запись
Ожидаемый результат
Все события зафиксированы
Изменение записей невозможно
Журнал является неизменяемым
TC-09 — Конфликт повторного въезда
Цель

Проверка обработки повторного въезда одного автомобиля.

Предусловия
Автомобиль уже находится на парковке
Шаги
Автомобиль повторно подъезжает к въезду
Камера фиксирует номер
Система проверяет статус автомобиля
Ожидаемый результат
Въезд блокируется
Пользователь получает предупреждение
Дублирующая запись не создаётся
TC-10 — Ручное открытие шлагбаума
Цель

Проверка ручного режима управления шлагбаумом.

Предусловия
Автоматическая система распознавания недоступна
Шаги
Оператор открывает интерфейс управления
Выбирает команду «Открыть шлагбаум»
Подтверждает действие
Ожидаемый результат
Шлагбаум открывается вручную
Событие фиксируется с причиной «manual override»
TC-11 — Оффлайн режим приложения
Цель

Проверка работы мобильного приложения без интернет-соединения.

Предусловия
Пользователь авторизован
Интернет отключён
Шаги
Пользователь завершает сессию в оффлайн-режиме
Данные сохраняются локально
Интернет восстанавливается
Происходит синхронизация
Ожидаемый результат
Данные не теряются
После восстановления сети выполняется синхронизация
Система корректно обновляет статус
TC-12 — Ошибка распознавания номера
Цель

Проверка обработки ошибки LPR.

Предусловия
Камера не может распознать номер
Шаги
Автомобиль подъезжает к шлагбауму
Камера не распознаёт номер
Система не находит совпадение
Ожидаемый результат
Шлагбаум не открывается
Событие фиксируется как ошибка распознавания
Уведомляется оператор
TC-13 — Уведомления о завершении парковки
Цель

Проверка системы уведомлений.

Шаги
Пользователь завершает парковочную сессию
Система формирует уведомление
Отправляет push-уведомление
Ожидаемый результат
Уведомление доставляется ≤ 10 секунд
Пользователь получает сообщение об оплате/завершении
TC-14 — Нагрузочное тестирование (2000 автомобилей)
Цель

Проверка устойчивости системы при высокой нагрузке.

Шаги
Эмулируется одновременный въезд 2000 автомобилей
Система обрабатывает события параллельно
Фиксируются задержки и ошибки
Ожидаемый результат
Система остаётся стабильной
Потерь событий нет
Время обработки соответствует требованиям NFR
# Критерии готовности

- Все требования покрыты тестами  
- ≥ 12 тест-кейсов  
- Есть негативные сценарии  
- Есть нагрузочное тестирование  
- Есть деградационные режимы  
- Есть полная трассируемость
