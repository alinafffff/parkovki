# 7. Верификация и валидация (V&V) — ParkPoint

## Обозначения

### Метод проверки:
- **T** — тест (функциональный, интеграционный, нагрузочный)
- **A** — анализ (метрики, логи, отчёты)
- **I** — инспекция (архитектура, конфигурация, БД, UI)
- **D** — демонстрация (сквозной сценарий)

# 7.2 Тест-кейсы

---

## TC-01 — Въезд автомобиля через распознавание номера

### Цель
Проверить автоматический въезд автомобиля через камеру LPR (License Plate Recognition).

### Предусловия
- Камера распознавания номеров активна  
- Пользователь зарегистрирован в системе  
- Авто привязано к аккаунту  

### Шаги
1. Автомобиль подъезжает к шлагбауму  
2. Камера фиксирует номер  
3. Система проверяет номер в базе  
4. При совпадении отправляется сигнал на шлагбаум  
5. Шлагбаум открывается  
6. Записывается событие въезда  

### Ожидаемый результат
- Шлагбаум открыт  
- В БД создана запись: номер авто + время въезда  
- Пользователь отображается как "на парковке"  

---

## TC-02 — Расчёт стоимости парковки

### Цель
Проверить корректность расчёта стоимости.

### Предусловия
- Авто уже на парковке  
- Тариф: 100 руб/час  

### Шаги
1. Пользователь завершает парковку  
2. Система фиксирует время выезда  
3. Вычисляется длительность  
4. Рассчитывается стоимость  
5. Отображается сумма к оплате  

### Ожидаемый результат
- Формула применена корректно  
- Сумма соответствует времени (час × тариф)  

---

## TC-03 — Оплата парковки через приложение

### Цель
Проверить оплату через мобильное приложение.

### Предусловия
- Есть неоплаченная парковка  
- Подключён платежный шлюз  

### Шаги
1. Пользователь открывает приложение  
2. Выбирает активную сессию  
3. Нажимает "Оплатить"  
4. Подтверждает оплату  
5. Платежный шлюз возвращает статус  

### Ожидаемый результат
- Статус = "Оплачено"  
- Сессия закрыта  
- Пользователь получает уведомление  

---

## TC-04 — Резервирование парковочного места

### Цель
Проверить возможность бронирования места.

### Шаги
1. Пользователь выбирает парковку  
2. Выбирает время  
3. Нажимает "Забронировать"  
4. Система резервирует место  

### Ожидаемый результат
- Место закреплено  
- В системе отображается статус "reserved"  

---

## TC-05 — Контроль занятости мест

### Цель
Проверить обновление занятости в реальном времени.

### Шаги
1. Несколько машин заезжают  
2. Камеры фиксируют занятость  
3. Система обновляет карту парковки  

### Ожидаемый результат
- Карта обновляется без задержек  
- Кол-во свободных мест корректно  

---

## TC-06 — Деградация при отключении платежного шлюза

### Цель
Проверить работу без внешней оплаты.

### Шаги
1. Отключить платежный шлюз  
2. Пользователь завершает парковку  
3. Система переводит оплату в "ожидание"  

### Ожидаемый результат
- Сессия не теряется  
- Оплата отложена  
- Уведомление пользователю отправлено  

---

## TC-07 — Управление тарифами администратором

### Цель
Проверить изменение тарифов.

### Шаги
1. Администратор открывает панель  
2. Меняет тариф  
3. Сохраняет изменения  

### Ожидаемый результат
- Новый тариф применяется ко всем новым сессиям  

---

## TC-08 — Журнал событий (audit log)

### Цель
Проверить неизменяемость логов.

### Шаги
1. Выполняются действия (въезд, выезд, оплата)  
2. Проверяется журнал  
3. Попытка изменения записи  

### Ожидаемый результат
- Изменение запрещено  
- Лог защищён  

---

## TC-09 — Конфликт повторного въезда

### Цель
Проверить защиту от двойного въезда.

### Шаги
1. Авто уже на парковке  
2. Повторный въезд  
3. Система проверяет статус  

### Ожидаемый результат
- Въезд отклонён  
- Предупреждение пользователю  

---

## TC-10 — Ручное открытие шлагбаума

### Цель
Проверить fallback режим.

### Шаги
1. Камера не работает  
2. Оператор открывает шлагбаум вручную  

### Ожидаемый результат
- Въезд зафиксирован вручную  
- Лог содержит причину  

---

## TC-11 — Оффлайн-режим приложения

### Цель
Проверить работу без интернета.

### Шаги
1. Отключить интернет  
2. Пользователь завершает сессию  
3. Данные сохраняются локально  
4. Синхронизация позже  

### Ожидаемый результат
- Данные не теряются  
- Синхронизация проходит позже  

---

## TC-12 — Распознавание номера (негативный сценарий)

### Цель
Проверить ошибку распознавания.

### Шаги
1. Камера не распознала номер  
2. Система отклоняет въезд  

### Ожидаемый результат
- Шлагбаум не открывается  
- Запрос у оператора  

---

## TC-13 — Уведомления о завершении парковки

### Цель
Проверить push-уведомления.

### Шаги
1. Пользователь завершает парковку  
2. Система отправляет уведомление  

### Ожидаемый результат
- Push приходит ≤ 10 секунд  

---

## TC-14 — Нагрузка системы (2000 авто)

### Цель
Проверить масштабируемость.

### Шаги
1. Симулировать 2000 въездов  
2. Проверить обработку событий  

### Ожидаемый результат
- Система стабильна  
- Нет потерь событий

# Критерии готовности

- Все требования покрыты тестами  
- Есть минимум 12 тест-кейсов  
- Есть негативные сценарии  
- Есть нагрузочное тестирование  
- Есть деградационные режимы  
- Есть traceability матрица
