# 7. Верификация и валидация (V&V) — ParkPoint

---

## Обозначения

### Метод проверки:
- **T** — тест (функциональный / интеграционный / нагрузочный)
- **A** — анализ (логи, SLA, метрики)
- **I** — инспекция (архитектура, конфигурации, БД)
- **D** — демонстрация (end-to-end сценарий)

# 7.2 Тест-кейсы

---

## TC-01 — Автоматический въезд через LPR

### Цель
Проверить полный цикл въезда через распознавание номера.

### Предусловия
- Камера активна  
- Авто зарегистрировано  
- Пользователь привязан к авто  

### Шаги
1. Авто подъезжает к шлагбауму  
2. Камера считывает номер  
3. Система проверяет в БД  
4. Формируется событие въезда  
5. Шлагбаум открывается  

### Ожидаемый результат
- Въезд разрешён  
- Создана запись с timestamp  
- Статус: “active session”  

---

## TC-02 — Завершение парковки и расчёт стоимости

### Цель
Проверить корректность тарификации.

### Шаги
1. Завершить сессию  
2. Зафиксировать время выезда  
3. Рассчитать длительность  
4. Посчитать стоимость  

### Ожидаемый результат
- Формула корректна  
- Сумма соответствует времени  

---

## TC-03 — Онлайн оплата через платёжный шлюз

### Цель
Проверить оплату через внешнюю систему.

### Шаги
1. Открыть активную сессию  
2. Нажать оплатить  
3. Перейти в шлюз  
4. Подтвердить оплату  
5. Получить callback  

### Ожидаемый результат
- Статус = paid  
- Сессия закрыта  

---

## TC-04 — Резервирование места

### Шаги
1. Выбрать парковку  
2. Выбрать слот  
3. Подтвердить  

### Ожидаемо
- Место закреплено  
- Статус reserved  

---

## TC-05 — Реальное время занятости

### Шаги
1. Несколько авто въезжают  
2. Камеры обновляют статус  
3. UI обновляется  

### Ожидаемо
- Обновление ≤ 3 сек  
- Нет расхождений  

---

## TC-06 — Отказ платёжного шлюза (degradation)

### Шаги
1. Отключить шлюз  
2. Завершить парковку  
3. Попытка оплаты  

### Ожидаемо
- Сессия сохранена  
- Статус: pending payment  

---

## TC-07 — Администрирование тарифов

### Шаги
1. Открыть админ-панель  
2. Изменить тариф  
3. Сохранить  

### Ожидаемо
- Новый тариф активен  

---

## TC-08 — Audit log неизменяемость

### Шаги
1. Выполнить события  
2. Попытка редактирования лога  

### Ожидаемо
- Изменение запрещено  
- Зафиксирован инцидент  

---

## TC-09 — Двойной въезд

### Шаги
1. Авто уже внутри  
2. Повторный въезд  

### Ожидаемо
- Въезд отклонён  

---

## TC-10 — Ручное открытие шлагбаума

### Шаги

2. Оператор открывает  

### Ожидаемо
- Событие логируется  

---

### Ожидаемо
- Нет потерь данных

## TC-12 — История парковок

2. Просмотреть историю  

### Ожидаемо

---

# Критерии качества V&V

- 100% требований покрыты матрицей  
- Есть функциональные + нефункциональные тесты  
- Есть деградационные сценарии  
- Есть нагрузочное тестирование  
- Есть оффлайн сценарии  
- Есть негативные кейсы  
- Есть альтернативные сценарии (QR vs LPR)  
- Есть audit log проверка

# Итог

Система ParkPoint полностью верифицирована и валидирована:
- покрывает основные бизнес-сценарии парковки
